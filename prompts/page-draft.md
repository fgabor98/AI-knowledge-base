# Prompt: Page Draft

Use this prompt to generate a first draft from an approved outline.

```text
Write a Markdown documentation page for the topic: <TOPIC>.

Use this exact structure:

# Topic name

## What problem does this solve?
## Core concepts
## Mental model
## Syntax / API / mechanism
## Minimal example
## Real-world example
## Common mistakes
## Debugging checklist
## Related topics
## References

Requirements:
- Keep the page practical and technically precise.
- Use concrete examples.
- Include commands where relevant.
- Include common mistakes and how to recognize them.
- Avoid vague claims.
- Do not invent references.
- Mark uncertain details explicitly as things to verify.
- Assume the page starts as an unreviewed draft.
```
